# FoxSay 产品边界

## 核心承诺
FoxSay 是以课程为核心的学习 Copilot。产品应该让人感觉是一只聪明、有点调皮的小狐狸——它比学生更懂当前课程,同时诚实地标明哪些内容来自课程材料、哪些是补充说明。

## 课程为原子单位
- 课程是材料、向量检索、骨架生成、聊天、复习计划的隔离边界。
- 跨课程检索暂不实现,除非未来产品需求明确引入。
- 所有关于材料、问题、回答、骨架、复习计划、`/btw` 插话的后端契约都包含 `course_id`。

## MVP 范围内
- 从 CSV/Excel 课程表导入,生成课程卡片和考试倒计时。
- 手动创建课程。
- 上传并注册材料,如 PDF、PPT、图片、文本备注。
- 异步处理材料,产出 chunks、embeddings、课程骨架。
- 课程边界内问答,走 CRAG 策略并带引用。
- 超级备考模式,包括复习计划生成、引导式复习、`/btw` 插话。

## 优先级原则
新增功能应有具体的用户需求支撑,优先服务于核心功能的完善和体验提升。避免在没有明确需求的情况下引入额外功能模块。

## 产品语气
- 文案简洁、自信、像狐狸。
- 材料不足时诚实说明,给出有用信息的同时标注来源可信度。
- 不要让 UI 看起来像通用企业聊天机器人。
