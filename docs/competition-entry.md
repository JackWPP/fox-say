## 【学习工作赛道】狐说 FoxSay —— 课程级 AI 学习 Copilot

### 1. 创意名称 + 创意介绍

**狐说（FoxSay）** 是一只"啃完一门课，带你过期末"的 AI 小狐狸。

我们发现，越来越多的大学生在期末复习时依赖 AI 工具，但传统的通用 ChatBot 存在三个致命问题：

- **幻觉问题**：对于课程细节，AI 会一本正经地胡说八道，学生无法辨别真伪
- **缺乏专业知识**：每所学校的课程体系、教师重点、考试风格各不相同，通用 LLM 的预训练语料不覆盖这些"微观知识"
- **复习资料碎片化**：PPT 在网盘、板书在 QQ 群、重点在老师口头划的、往年题在学长手里——这些信息从未被整合成一个可检索的知识体系

**FoxSay 的解法**：以"课程"为原子单位，把学生上传的所有材料（PDF、PPT、笔记、板书照片）消化成一张完整的**课程知识图谱**，然后在这个严格的课程边界内回答每一条问题——**每条回答带来源引用，超范围的问题主动拒答**。

这不是另一个通用问答机器人。这是一个**只在你的课程范围内，比你更懂这门课**的学习搭子。

**核心差异化机制——CRAG 边界控制**：

```
用户提问 → 向量检索课程材料 → 置信度评估
  ├── score ≥ 0.72  → 正常回答 + 来源引用（"来自 [线性代数PPT] · 第3章"）
  ├── 0.55 ≤ score < 0.72  → 谨慎回答，标注置信状态
  └── score < 0.55  → "这个问题超出了线性代数的范围，我不知道。"
```

**技术架构一句话**：PDF/PPT → 文档结构图（DMAP）→ 4 阶段 LangGraph 知识构建 → 7 工具 Agent ReAct 循环 → 三层混合检索 + CRAG 门控。

### 2. 目标用户及痛点

**面向用户**：中国在校本科生，尤其是期末复习"临时抱佛脚"的同学。

**使用场景**：
- **日常学习**：上课没听懂、写作业卡壳时，在课程框架内快速提问，不超纲
- **期末突击**：考前一周，把整学期材料扔进来，AI 消化后生成知识图谱 + 复习计划 + 陪伴复习

**当前痛点**：

| 痛点 | 现状 | FoxSay 的解法 |
|------|------|---------------|
| 复习资料无法形成知识体系 | PPT、笔记、板书散落各处 | 上传后自动构建课程知识图谱 + 骨架图 |
| AI 幻觉导致学到错误知识 | ChatGPT 会编造公式和定理 | CRAG 边界控制：只基于你的材料回答，超范围拒答 |
| 无法高效利用 AI 赋能复习 | 通用工具不懂你的课程体系 | 课程级 Agent：7 个专用工具，沿先修链路回答 |
| 复习缺乏结构和节奏 | 不知道先看什么、重点在哪 | 超级备考模式：自动生成复习计划 + 陪伴复习 + /btw 插话 |

---

### 3. 价值与意义

**效率提升**：传统期末复习，学生需要花 2-3 天整理资料、画思维导图、判断重点。FoxSay 把这个过程压缩到**材料上传后的几分钟内**——AI 自动消化材料、生成课程骨架图、标注薄弱环节。学生从"不知道从哪开始"直接进入"有结构地复习"。

**学习质量保障**：每条回答强制带来源引用，学生可以立刻验证 AI 说的是不是来自自己的材料。超范围的问题主动拒答，杜绝"AI 编的公式你背了一晚上"的悲剧。这不是让 AI 替你学，是让 AI 在你的课程边界内**帮你学得更准**。

**社会价值**：教育资源不均衡是中国高校的长期问题。重点高校有完善的辅导体系和学长资源，普通高校的学生往往只能靠自己。FoxSay 让每个学生都能拥有一个"懂这门课的学习搭子"——不挑学校、不挑老师，只要你有材料，它就能帮你建立知识体系。

### 4. 产品形态与交互亮点

**多端覆盖**：Web 端（浏览器即用）+ 未来扩展至手机/平板，成为常驻在设备里的学习搭子。

**狐狸人设**：不是冷冰冰的工具，是一只贱贱的、但极其聪明的小狐狸。它会说"你来了。期末了？还是还没死心想好好学？"，会主动诊断"这门课你最薄的地方好像是第三章"。

**关键交互体验**：
- **第一个惊喜**：材料处理完成后，不是"处理完成"通知，而是狐狸主动推送课程骨架图 + 一句话诊断
- **零摩擦捕获**：老师的口头暗示、学长的经验、自己的疑问，随手扔进来，系统异步归位
- **/btw 插话**：在主线复习过程中随时插问，快速回答后自动返回主线，不打断心流
- **诚实拒答**：超出课程范围的问题，狐狸直接说"不知道"，不编造、不泛化

### 5. 技术亮点

| 技术点 | 实现 |
|--------|------|
| 课程知识构建 | 4 阶段 LangGraph Pipeline（Supervisor → Workers → Reducer → Reviewer） |
| 检索增强生成 | 三层混合检索（章节级 / 知识点级 / 全量合并）+ CRAG 门控 |
| Agent 工具链 | 7 工具 ReAct 循环：搜索知识库、查询概念、追溯先修链路、获取来源原文等 |
| 知识图谱可视化 | React Flow 渲染，节点颜色区分重要程度，支持交互探索 |
| 增量更新 | Merkle Tree diff，新材料只处理增量部分，不全量重建 |
| 评测体系 | 自动生成 30 题评测集 + Qwen3.5-9B Faithfulness Judge 独立打分 |
| 文档解析 | pdfplumber（PDF）+ python-pptx（PPT），保留文档结构层级 |
